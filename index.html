What is Solvigil?

Solvigil is a smart contract security analyzer that scans Solidity (.sol) files to detect vulnerabilities, explain them clearly, simulate attacks, and suggest fixes. It makes blockchain security understandable and accessible to developers of all levels.

Rule-Based Detection

Advanced pattern-matching logic detects common vulnerabilities like reentrancy, access control issues, overflow/underflow, and more.

Clear Explanations

Every vulnerability comes with detailed explanations, severity levels, vulnerable code highlighting, and step-by-step fix recommendations.

Attack Simulation

Understand the attack flow with simulated exploits and real-world examples. See exactly how vulnerabilities can be abused.

How It Works

01

Upload Your Code

Drag and drop your .sol file or paste your Solidity code directly into the interface.

02

Parse & Analyze

Code is converted into an Abstract Syntax Tree (AST) and analyzed with rule-based detection logic.

03

Detect Vulnerabilities

Detection rules identify security issues including reentrancy, access control, unchecked calls, integer overflow/underflow, and more.

04

Get Results & Fix

Receive detailed report with severity levels, code highlighting, attack simulations, and recommended fixes.

Vulnerabilities We Detect

Reentrancy

Critical

External calls before state updates allow attackers to manipulate state across multiple contexts.

Access Control Issues

High

Missing ownership checks or improper permission validation allowing unauthorized actions.

Unchecked External Calls

High

No validation of call results can hide failures from the contract execution flow entirely.

Integer Overflow/Underflow

Critical

Unsafe arithmetic operations without proper validation (primarily found in pre-Solidity 0.8).

Solidity Version Risks

Medium

Using outdated or deprecated Solidity versions mapped to historically known vulnerabilities.

Denial of Service (DoS)

High

Attacks and logic blocks that can freeze, exhaust gas, or permanently halt contract functionality.

Key Concepts

Vulnerability

A weakness or flaw in smart contract code that can be exploited to cause unintended behavior, leading to loss of funds, unauthorized access, or system failure.

Static Analysis

Automated examination of Solidity code without executing it, identifying potential vulnerabilities by analyzing patterns, logic flows, and security best practices.

Attack Simulation

Step-by-step walkthrough of how a vulnerability can be exploited, demonstrating the attack flow and impact on the contract directly to developers.

Abstract Syntax Tree (AST)

A tree representation of the structure of code, used by the parser to understand code syntax and semantics for precise and highly accurate analysis.
